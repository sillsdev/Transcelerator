Overview of Basic Features

File

  • saveSave on demand or Auto Save saves every time a translation is changed (on by default)
  • refreshReload forces a re-analysis of the questions to improve guessed translations (after manual editing or addition of questions)
  • gradingGenerate Checking Scriptproduces an HTML-format script by retrieving Scripture portions from Paratext and interleaving them with the relevant questions. The script can be customized in various ways. Once produced, it can be printed or shared electronically.
  • Produce Scripture Forge Files saves translated questions in a format that can be used by Scripture Forge. These files will be automatically synchronized when using Send/Receive in Paratext. Note: Icon displays green when this is toggled on.

Editing/Navigation

On the Edit menu, select Previous Untranslated Question and Next Untranslated Question to go to the next question that does not yet have a confirmed translation. When editing a translation use Shift Word(s) Right and Shift Word(s) Left to compare_arrowsreorder the current or selected word. On the toolbar there are also two buttons that allow the user to control whether to send () and receive () Scripture references to keep Transcelerator in sync with other applications (e.g., Paratext) when navigating.

Question Customization

Since there are limitations to using a “canned” set of questions to check comprehension, Transcelerator allows the addition of custom questions and the exclusion or modification of existing source questions as needed. The command for customizing the source questions are accessible using the context menu in the main grid:

  • thumb_down_altExclude Question that is not useful for the intended audience.
  • thumb_up_altInclude Question that was previously excluded (see all_inclusiveviewing excluded questions).
  • editEdit Question to select an alternate form or manually alter the wording.
  • addAdd a New Question

filter_altFiltering and Sorting

Transcelerator allows the list of questions to be filtered in the following ways:

  • By reference range
  • Textual
  • Based on whether or not biblical terms have renderings (i.e., vernacular equivalents)

In addition, the questions can be sorted by any of the columns by clicking the column header. Click the header of the sorted column a second time to reverse the sort order.

View

To eliminate unnecessary elements of the user interface (especially helpful when working on a small display) the following items can be hidden:

  • Generated Translation Details - Provides technical analysis details mainly useful for understanding how guessed translations are being determined.
  • Edit Question Column - This column is useful when customizing source questions. (Hidden by default.) See Editing Questions for more information.
  • Answers and Comments - Often helpful when translating questions, but not strictly necessary.
  • Toolbar - Shows the box for textual filtering, the synchronized navigation buttons and some status information.
  • Biblical Terms Pane - Makes it easy to select the correct rendering for biblical terms used in a question.

all_inclusiveIf questions have been excluded, they can be displayed (primarily so they can be re-included if so desired) by selecting Excluded Questions.

zoom_inUse Zoom In (Ctrl + Plus) or Zoom Out (Ctrl + Minus) to increase or decrease the size of the non-vernacular text in the main grid. Note: The font and size of the vernacular text displayed in the Translation column is determined by the font settings for the vernacular language in Paratext.

languageTranscelerator can display the source questions and the user interface in any locale that was included when it was installed. Select the desired language from the Display Language menu.
